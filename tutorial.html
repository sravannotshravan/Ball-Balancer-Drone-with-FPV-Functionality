What the app does

The project is a browser-based drone simulation. You control a drone, keep a ball balanced in a tray, and use assist features when the flight becomes unstable. The desktop layout focuses on telemetry and control hints. The mobile layout focuses on front-camera viewing and stereo FPV mode.

Project map

index.html

Desktop entry page. It shows the front camera frame, the HUD, telemetry, and control hints.

mobile.html

Mobile entry page. It keeps the front camera first and supports the stereo split layout.

main.js

Main simulation loop, camera setup, HUD updates, assist logic, environment switching, and crash handling.

drone.js

Builds the drone model and tray assembly used by the physics simulation.

input.js

Handles keyboard and gamepad input, including the toggles for assist and environment mode.

server.js

Static file server that serves the app on port 4173.

style.css

Styles the HUD, preview frame, overlays, and stereo mobile layout.

Dockerfile and scripts

Launch helpers for Docker, Windows, macOS, and Linux.

How the interface works

  1. Canvas world: Three.js renders the simulation while the UI floats over the scene.
  2. Front camera frame: The small camera box in the corner mirrors the forward view.
  3. Compass and telemetry: These panels report heading, altitude, speed, environment, tray state, and center status.
  4. Control strip: Shows whether keyboard or gamepad input is active and lists the current hotkeys.
  5. Debug HUD: F3 toggles the extended panel with full control instructions and extra telemetry values.
  6. Overlay: When the ball leaves the tray or the drone crashes, the overlay appears with reset instructions.
  7. Mobile stereo mode: On mobile, double tap toggles the split FPV layout for a VR-style view.

Controls

Windows and keyboard

  • W / A / S / D: Move the drone.
  • Arrow keys: Pitch and roll.
  • B: Toggle auto-balance.
  • G: Toggle assisted mode.
  • T: Switch environment.
  • F3: Toggle debug HUD.

Gamepad

  • Left stick: Altitude and yaw.
  • Right stick: Pitch and roll.
  • Y: Toggle assisted mode.
  • X: Toggle environment.
  • B: Toggle auto-balance.

Runtime flow

  1. Browser loads index.html or mobile.html.
  2. main.js creates the Three.js scene and the Cannon physics world.
  3. Input is read every frame, then the drone and tray update based on the active mode.
  4. The HUD and overlays reflect the current state of the ball, tray, and environment.
  5. The renderer draws the scene and keeps the front camera and mobile view in sync.
